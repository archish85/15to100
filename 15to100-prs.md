## PR 1 — Answer Feedback Flash (biggest engagement win)

**Problem:** Answering a question silently turns the bubble to a checkmark with no reaction. No dopamine hit.

**File:** `src/components/QuestionCard.jsx`

**Change:** Add a `flashResult` state that triggers a green/red border glow + banner for 1.2 seconds after answering.

```jsx
// src/components/QuestionCard.jsx
// ADD this import at the top if not already present:
import React from 'react';

const QuestionCard = ({ question }) => {
  const [flashResult, setFlashResult] = React.useState(null); // 'correct' | 'wrong' | null
  const { answerQuestion } = useGameStore();

  // REPLACE your existing handleAnswer with this:
  const handleAnswer = (optionIndex) => {
    if (question.answered) return;
    const isCorrect = question.answer === optionIndex;
    setFlashResult(isCorrect ? 'correct' : 'wrong');
    setTimeout(() => setFlashResult(null), 1200);
    answerQuestion(question.question_id, optionIndex);
  };

  // REPLACE your outer wrapper div's className with this:
  // (find the div that wraps the whole card — typically has border-slate-700)
  return (
    <div className={`
      bg-slate-800/60 rounded-xl p-6 border transition-all duration-300 relative
      ${flashResult === 'correct'
        ? 'border-green-400 shadow-[0_0_24px_rgba(74,222,128,0.4)]'
        : flashResult === 'wrong'
        ? 'border-red-400 shadow-[0_0_24px_rgba(248,113,113,0.4)]'
        : 'border-slate-700'}
    `}>

      {/* ADD this flash banner right inside the wrapper div, before everything else */}
      {flashResult && (
        <div className={`
          absolute top-0 left-0 right-0 rounded-t-xl py-2 text-center
          font-bold text-sm tracking-wide z-20
          animate-in fade-in duration-150
          ${flashResult === 'correct' ? 'bg-green-500 text-white' : 'bg-red-500 text-white'}
        `}>
          {flashResult === 'correct'
            ? `✓ Correct! +${question.difficulty === 'Purple' ? 15 : question.difficulty === 'Orange' ? 9 : 3} pts`
            : '✗ Wrong — no points'}
        </div>
      )}

      {/* rest of your existing card JSX unchanged */}
    </div>
  );
};
```

## PR 2 — Score Panel: Replace "-100" with Progress Framing

**Problem:** The score panel shows `-100` in large yellow text on first load. It looks like an error, not a game mechanic. Players need to understand they are 100 points *away* from winning, not that they have -100.

**File:** `src/components/ScorePanel.jsx`

**Change:** Replace the raw score number with a progress bar + "X to go" framing. Keep the CountUp animation.

```jsx
// src/components/ScorePanel.jsx — FULL REPLACEMENT

import React from 'react';
import CountUp from 'react-countup';

const ScorePanel = ({ score }) => {
  // score starts at -100 and goes to 0 (win)
  // pointsToGo: how far from winning (100 at start, 0 when won)
  const pointsToGo = Math.max(0, -score);
  const progress = Math.min(100, ((100 - pointsToGo) / 100) * 100); // 0% to 100%
  const isWon = score >= 0;

  return (
    <div className="bg-white/80 dark:bg-slate-900/80 backdrop-blur-md rounded-xl p-6 border border-yellow-500/30 flex flex-col items-center justify-center shadow-sm min-h-[140px] transition-colors duration-300">

      {/* Main number: show points remaining, not negative score */}
      <div className={`text-6xl font-black mb-1 tracking-tighter ${isWon ? 'text-green-500' : 'text-yellow-400'}`}>
        {isWon ? '🎉' : (
          <CountUp
            end={pointsToGo}
            duration={1.5}
            preserveValue={true}
          />
        )}
      </div>

      <h2 className="text-slate-500 dark:text-gray-400 text-sm font-bold uppercase tracking-wider mb-3">
        {isWon ? 'You reached 100!' : 'Points to 100'}
      </h2>

      {/* Progress bar */}
      {!isWon && (
        <div className="w-full bg-slate-700 rounded-full h-2 overflow-hidden">
          <div
            className="bg-yellow-400 h-2 rounded-full transition-all duration-700 ease-out"
            style={{ width: `${progress}%` }}
          />
        </div>
      )}
    </div>
  );
};

export default ScorePanel;
```

## PR 6 — Wrong Answer Shows Correct Answer

**Problem:** When you answer wrong, there's no feedback about what the right answer was. Players learn nothing and feel confused.

**File:** `src/components/QuestionCard.jsx`

**Change:** After a wrong answer, highlight the correct option in green while the wrong one turns red. This requires reading `question.answered` and `question.correct` from the question state.

```jsx

  if (question.answered) {
    if (index === question.answer) {
```
